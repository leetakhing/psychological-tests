
## BM25 Memory System Implementation

**Status**: ✓ PRIMARY SEARCH METHOD (全面切换)

**2026-02-08 Updates:**
- Switched to BM25 as primary memory search
- Created convenience script: `bin/bm25-search`
- Updated AGENTS.md with BM25 search workflow
- Added cron for daily incremental index updates (2 AM)
- Notion backup format: Parent page + child pages (B方案)

**Usage:**
```bash
# Quick search
bm25-search "Leo 的狗"

# Build/update index
node scripts/index-memory.js build --incremental
```

**Benefits:**
- Zero API calls (token savings)
- ~10ms response time
- Works without OpenAI/Google API keys

**Problem**: 
- OpenClaw memory_search needs API key (OpenAI/Google) - not available
- Python 3.6.8 too old for rank-bm25 library (needs 3.8+)
- 2GB RAM server, need memory-efficient solution

**Solution**: Pure Node.js BM25 implementation

**Implemented**:
- `scripts/index-memory.js` - Build/update BM25 index
- `scripts/search-memory.js` - Search CLI
- `SKILL.md` - Full documentation
- Auto tokenization (English regex + Chinese bigram fallback)
- Incremental update support

**Commands**:
```bash
# Build index
node scripts/index-memory.js build

# Search
node scripts/search-memory.js "Leo 的狗"

# Incremental update
node scripts/index-memory.js build --incremental
```

**Index Format**: JSON file (~1MB for 2 memory files)
**Search Speed**: ~10ms
